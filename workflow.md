# Migration Workflow

## Phase 1 - Preparation for Migration
- Provided migration engineer with environment details including number of users, domain information, mail service info, proxy/firewall status, OS distribution, POP usage, and file sharing structure.
- Receive preparation list such as mail backup, os updates, 
- Selected the license that will be used, Business Premium for all users.
- Prepare user guide for migration process. Win/Mac OS update, mail backup/load and Office365 update guide 

## Phase 2 - Backup & Preparation
- Assisted users to create PST/OLM backups using Outlook classic/legacy mode depending on their OS. 
- Users who have large size of mail backup, assisted them to use portable SSD to prevent data loss. 
- Also sent notification to prepare migrating files in OneDrive of old tenant to new tenant, specifically SharePoint 

## Phase 3 - Tenant Setup
- Used remote access tool with migration engineer to configure tenant. 
- With Domain service provided, added domain into new tenant for preparation. 
- Created user accounts for all users and assigned roles depending on positions, and licenses. 

## Phase 4 - Migration & DNS settings
- Remote access with engineer, changed DNS record to use M365 exchange server for emails. 
- After configuration, Verified mail flow for both individual accounts and group mailboxes.

## Phase 5 - Post-Migration Support
- Assisted with logins related to password, MFA, and Outlook settings.
- Guided user with how to use SharePoint and using them through teams synchronization. 
- Added meeting rooms to be selected on schedule. 
- Verified group mail setting for Send as and Delegate for users.
